VibeOps

The relay I built after getting tired of being the relay myself.

RoleSolo Developer
StackLangGraph · Claude API
Status8/8 Tests Passing, CI
The Problem

Working across multiple AI tools for a single task means constantly copy-pasting context between them — what one agent decided, what it tried, what failed. That manual relay work is exactly the kind of thing that should be automated, and it's a real bottleneck once a workflow needs more than one specialized step.

The Approach

VibeOps is a multi-agent orchestrator: a graph of specialized agents (built with LangGraph, powered by the Claude API) that pass context to each other automatically instead of through a human intermediary.

  • Shared memory layer across agents using SQLite (structured state) and ChromaDB (semantic/vector recall)
  • Retry and failure handling built into the graph, so a single agent failure doesn't collapse the whole pipeline
  • A full test suite — 8/8 passing — covering the orchestration logic, not just happy-path demos
  • CI via GitHub Actions, so every push is verified automatically before it's trusted
  • A Streamlit front end for demoing the pipeline interactively

Outcome & Honest Limits

The orchestration layer is solid — tested, CI-verified, and handles agent failure gracefully. What it doesn't yet do: dynamically add new agent types without a code change, and the ChromaDB recall is tuned for short-to-medium context, not long-running sessions. Both are documented as known limits in the repo rather than glossed over.
